## Tech Stack
- Language: TypeScript strict mode throughout all packages and apps
- Package manager: pnpm (workspace-aware; required by Turborepo)
- Monorepo: Turborepo + pnpm workspaces; Nx rejected (enterprise overhead); pnpm-only rejected (no task caching)
- Frontend: Vite + React SPA; scaffold via `npx create-turbo@latest` + Vite react-ts template; Next.js rejected (SSR no benefit); Astro rejected (SSG/islands conflicts with SPA state)
- Styling: Tailwind CSS + shadcn/ui + Radix UI primitives; design tokens as Tailwind theme extensions in tailwind.config.ts
- State: Zustand (two separate stores); async fetching via React Router v6 loaders only
- Routing: React Router v6; routes: `/` (library), `/read/:storyId`
- Validation: AJV v8 (JSON Schema Draft-07); verify ESM import chain before writing validation (AJV v8 is CommonJS вЂ” may need `createRequire` shim)
- Package build: tsup (CJS + ESM outputs with exports map)
- Testing unit: Vitest + @testing-library/react
- Testing e2e: Playwright (Chromium + Firefox + WebKit; iPhone 14 viewport)
- Hosting: Vercel (zero config Vite SPA; preview deployments per PR; CDN; free tier)
- CI/CD: GitHub Actions вЂ” lint в†’ type-check в†’ test:unit в†’ test:e2e в†’ build on every PR; Vercel deploys on merge to main
- Font: Noto Sans JP (web font, subset for Genki kanji range) + Inter for UI + system CJK fallback

## Turborepo Pipeline
- build-vocab: inputs=[scripts/build-vocab.ts, scripts/data/genki-vocab.csv]; outputs=[apps/web/public/vocab.json]; cache=true
- build: dependsOn=["^build", "build-vocab"]; outputs=["dist/**"]
- dev: dependsOn=["^build", "build-vocab"]; persistent=true; cache=false
- typecheck: dependsOn=["^build"]
- test:unit: dependsOn=["^build", "build-vocab"]
- test:e2e: dependsOn=["build"]; cache=false
- `turbo dev` is the only supported dev entrypoint вЂ” not `pnpm dev` in apps/web directly
- test:unit and test:e2e are separate tasks to prevent Playwright webServer port collisions with Vitest

## Canonical Type Definitions (packages/schema/src/types.ts)
```ts
interface VocabEntry { id: number; word: string; reading: string; meaning: string; lesson: string; notes?: string }
interface KanjiEntry { character: string; meaning: string; onYomi: string[]; kunYomi: string[] }
interface StoryModel {
  schemaVersion: string; id: string; title: string; titleJa: string; language: string;
  difficulty: string | null; description: string; keywords: VocabSupplementEntry[];
  grammar: string[]; vocabSupplement: VocabSupplementEntry[];
  sentences: SentenceModel[]; metadata: Record<string, unknown>
}
interface SentenceModel {
  id: string; words: string[]; ruby: (string | null)[]; vocabKeys: (number | null)[];
  translation: string | null; grammar: number[] // indices into StoryModel.grammar
  audioUrl?: string // stored v1; not played; wire format: audio_url
}
interface VocabSupplementEntry { word: string; hiragana: string; translation: string }
type LookupState =
  | { status: 'idle' }
  | { status: 'found'; word: string; entry: VocabEntry }
  | { status: 'not-found'; word: string }
```
- CRITICAL: StoryModel.grammar = string[] (descriptions); SentenceModel.grammar = number[] (indices) вЂ” must not be conflated

## State Management
- Two Zustand stores, strictly separated вЂ” lookupStore never imports from preferenceStore and vice versa
- LookupStore interface: `{ lookupState: LookupState; selectedSentenceId: string | null; lookup(word, entry|null, sentenceId): void; selectSentence(sentenceId): void; reset(): void; _reset(): void }`
- Word tap в†’ `lookup(word, entry, sentenceId)` updates both lookupState AND selectedSentenceId
- Sentence container tap в†’ `selectSentence(sentenceId)` updates selectedSentenceId, resets lookupState to idle
- preferenceStore persisted keys: rubyVisible, spacingVisible, transVisible, textSize ('small'|'medium'|'large'), activeTab ('story'|'vocabulary'|'grammar')
- Selector pattern: `useShallow` for object selections; no useShallow needed for primitive selections
- Store actions: synchronous state updates ONLY; all async data fetching in React Router loaders (never useEffect, never store actions)

## Data Architecture
- Genki vocab CSV: scripts/data/genki-vocab.csv; preprocessed at build time via scripts/build-vocab.ts в†’ apps/web/public/vocab.json (gitignored); loaded via `fetch('/vocab.json')` в†’ in-memory Map keyed by integer id; O(1) lookup; no runtime CSV parsing
- CSV row IDs: immutable once assigned; CSV is append-only; column schema frozen before authoring tool development begins
- Kanji data: apps/web/public/kanji-data.json (committed); JSON object keyed by literal kanji character; loaded via `fetch('/kanji-data.json')` в†’ in-memory Map; O(1) lookup; no build step
- Story library manifest: apps/web/public/stories/manifest.json; fetched at startup via React Router loader; entries: {id (permanent slug), filename, title, titleJa, difficulty, language, description}
- Story ID format: kebab-case permanent slug (e.g. `genki-i-ch6-tanaka-letter`); permanent once assigned, never reused; route param `:storyId`
- Local upload persistence: UUID (client-generated) + IndexedDB; `/read/:uuid` works identically to library stories; local URLs not shareable by design
- Both vocab and kanji data fetched once at app startup into in-memory Maps вЂ” available synchronously for all subsequent lookups
- Supplement matching: exact token string match against StoryModel.vocabSupplement вЂ” no morphological normalisation

## Story Format Contract
- Schema file: packages/schema/schemas/story.v1.json (source of truth for nihonnohon loader and AI authoring tool)
- `additionalProperties: false` at every object node вЂ” prevents silent field drift
- CI validates all story fixture files against schema
- `schema_version`: string enum (e.g. `"1"`, `"2"`); integer-as-string; no minor/patch subdivision; every schema change gets new version + new loader
- Wire format: snake_case; TypeScript internal: camelCase; transformation in v1 loader only
- AJV validates snake_case wire format FIRST; transformation to camelCase StoryModel AFTER validation passes
- Schema changes must be coordinated between nihonnohon and AI authoring tool before version bump

## Story JSON Wire Format Fields
- Story-level: schema_version, id, title, title_ja, language, difficulty, description, keywords [{word, hiragana, translation}], grammar [strings], vocab_supplement [{word, hiragana, translation}], metadata {}
- Sentence-level: id (stable unique), words [strings, minLength:1], ruby [string|null], vocab_keys [integer|null], translation (optional string), grammar [number[] вЂ” indices into story grammar array], audio_url (optional string URI вЂ” stored, not played v1)
- Parallel array invariant: words, ruby, vocab_keys MUST have equal length per sentence; parser throws on mismatch

## Versioned Loader Architecture
- Dispatch: packages/story-loader/src/index.ts вЂ” version-keyed registry `const LOADERS = { "1": loadV1 }`
- Each loader: AJV validate в†’ transform to version-agnostic StoryModel в†’ return StoryModel OR throw LoaderError
- Unrecognised schema version в†’ LoaderError('UNSUPPORTED_VERSION') with user-facing message naming the version
- packages/story-loader/src/v1.ts is the ONLY code that knows story JSON wire format; everything upstream receives only StoryModel

## Route Resolution
- `/read/:storyId`: (1) manifest lookup by ID в†’ fetch from public/stories/; (2) IndexedDB lookup by UUID в†’ load local content; (3) neither в†’ "story not found" with link to library
- IndexedDB UUID-not-found error: LoaderError('PARSE_FAILED'); ErrorBoundary renders: "This story isn't available вЂ” it may have been loaded on another device or the data was cleared." + link to library
- vercel.json SPA rewrite required: `{ "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }` вЂ” without this, /read/:storyId returns 404 on hard refresh

## Data Flow
- Word tap: WordToken.onClick в†’ useLookupStore.lookup(word, vocabKey, sentenceId) в†’ vocabService.lookupVocab(vocabKey) в†’ lookupState 'found'|'not-found' в†’ InfoPanel renders; SentenceBlock selectedSentenceId match в†’ bg-accent-subtle highlight
- Library story open: ReaderRoute loader в†’ fetch manifest.json в†’ find storyId в†’ fetch /stories/{filename}.json в†’ loadStory(rawJson) в†’ v1 loader в†’ AJV validate в†’ transform в†’ StoryModel в†’ route data в†’ ReaderRoute renders
- Local file upload: LibraryRoute file input в†’ File API в†’ loadStory(rawJson) в†’ validate в†’ StoryModel в†’ indexedDbService.saveStory(uuid, rawJson) в†’ navigate('/read/:uuid') в†’ ReaderRoute loader в†’ IndexedDB hit в†’ loadStory() в†’ StoryModel

## Error Handling
- LoaderError('UNSUPPORTED_VERSION') в†’ React Router ErrorBoundary в†’ message + link to library
- LoaderError('SCHEMA_INVALID') в†’ inline error below upload trigger + link to format spec
- LoaderError('PARSE_FAILED') в†’ inline error below upload trigger
- null from vocabService в†’ lookupState {status:'not-found'} в†’ "No entry for [word]"
- null from kanjiService в†’ KanjiBreakdown section not rendered
- null vs LoaderError are distinct failure modes вЂ” never conflate

## Japanese Text Rendering Patterns
- `lang="ja"` on every element rendering Japanese text вЂ” no exceptions
- Ruby: `<ruby>/<rt>` with `visibility: hidden` toggle (never `display: none` вЂ” collapses rt space, causes reflow)
- Ruby container: `ruby { ruby-position: over; ruby-align: center; }` + `.word-token { white-space: nowrap; }`
- Spacing toggle: CSS `gap` with 150ms transition; `gap-x-2` (visible) / `gap-x-0` (hidden)
- Text size: CSS custom property `--story-font-size`; TEXT_SIZE_VALUES = { small:'1rem', medium:'1.25rem', large:'1.5rem' }

## Naming Conventions
- Files: PascalCase.tsx (component), usePascalCase.ts (hook), camelCase.ts (service/utility), camelCaseStore.ts (store), [filename].test.ts(x) co-located
- Code: PascalCase (components, types/interfaces), use+PascalCase (hooks), use+PascalCase+Store (store exports), camelCase verb-noun (service functions), UPPER_SNAKE_CASE (constants), kebab-case (CSS design tokens)
- No re-export barrel index.ts files вЂ” import source files directly

## Import Patterns
- Cross-package: `import { StoryModel } from '@nihonnohon/schema'`; `import { loadStory, LoaderError } from '@nihonnohon/story-loader'`
- Intra-app: `import { WordToken } from '@/components/WordToken'`; `import { useLookupStore } from '@/stores/lookupStore'`
- Path alias: `@/*` в†’ `./src/*` defined in both tsconfig.app.json AND vite.config.ts

## Styling Rules
- Tailwind utilities only; CSS custom properties for dynamic values only (`--story-font-size`)
- No custom CSS classes outside tailwind.config.ts token definitions; no inline style={{}} except CSS custom properties

## Code Placement Rules
- snake_caseв†’camelCase transform в†’ packages/story-loader/src/v1.ts ONLY
- Types shared across packages в†’ packages/schema/src/types.ts
- Types local to web app в†’ apps/web/src/types.ts
- Vocab/kanji lookup logic в†’ apps/web/src/services/
- Zustand store slice в†’ apps/web/src/stores/
- React component в†’ apps/web/src/components/
- Route-level component в†’ apps/web/src/routes/
- Pure utility (no side effects) в†’ apps/web/src/utils/
- Test fixtures/mock data в†’ __fixtures__/ co-located with test file

## Architectural Boundaries
- Format boundary: v1.ts only code knowing wire format; upstream receives only StoryModel
- Data boundary: vocabService/kanjiService own in-memory Maps; no component accesses vocab.json/kanji-data.json directly
- State boundary: lookupStore + preferenceStore are only shared state; React Router loaders own all async fetching
- Package boundary: apps/web imports compiled @nihonnohon/* outputs only вЂ” enforced by eslint-plugin-boundaries
- Schema boundary: story.v1.json is contract between nihonnohon loader and AI authoring tool; breaking change requires new schema version + new loader

## Test Patterns
- `afterEach(() => useLookupStore.getState()._reset())` in all tests using lookupStore
- `beforeEach(() => localStorage.clear())` in all tests using preferenceStore
- Co-locate __fixtures__/ with test files
- E2e golden path: open app в†’ see library в†’ tap story card в†’ reader loads в†’ read sentence в†’ tap word в†’ InfoPanel updates в†’ tap second word в†’ InfoPanel updates в†’ back to library
- E2e error paths: invalid story upload; unsupported schema version; manifest fetch failure; story ID not found; UUID not found
- E2e accessibility: axe-core + visual regression (toHaveScreenshot for toggle states)

## Anti-Patterns
- `import from '../../packages/schema/src/types'` вЂ” use @nihonnohon/schema
- `<rt style={{ display: 'none' }}>` вЂ” use `invisible` (visibility: hidden)
- `onKeyDown: e.key === 'Enter' only` вЂ” Space is required
- `StoryModel.grammar used as number[]` вЂ” it is string[]
- `fetch inside useEffect for story data` вЂ” use React Router loader
- `LookupState as { word: string | null }` вЂ” use discriminated union
- `import vocabData from '@/data/vocab.json'` вЂ” use vocabService
- `creating re-export barrel index.ts files` вЂ” import source files directly
