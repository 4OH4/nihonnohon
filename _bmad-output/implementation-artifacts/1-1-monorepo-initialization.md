# Story 1.1: Monorepo Initialization

Status: ready-for-dev

## Story

As a **developer**,
I want the Turborepo + pnpm monorepo scaffolded with all package directories in place,
so that every subsequent story has a valid, installable repository to build on.

## Acceptance Criteria

1. **Given** `npx create-turbo@latest nihonnohon` is run with pnpm selected as the package manager **When** scaffold completes **Then** repo root contains `turbo.json`, `package.json` (private: true), `pnpm-workspace.yaml`, and `.gitignore`; default scaffold apps are removed or replaced with project-specific stubs.

2. **Given** `pnpm-workspace.yaml` **When** reviewed **Then** lists: `apps/web`, `apps/api`, `packages/*`; `apps/story-generator` is explicitly NOT listed (Python runtime, excluded from pnpm workspace).

3. **Given** the package directories are created **When** reviewed **Then** the following exist as valid, installable TS package stubs (each with `package.json`, `tsconfig.json`, and an empty `src/index.ts`): `packages/schema/`, `packages/story-loader/`, `packages/typescript-config/`, `packages/eslint-config/`; `apps/web/` exists as a stub; `apps/api/` exists with a minimal `package.json` placeholder.

4. **Given** `pnpm install` is run from repo root **When** complete **Then** exits 0; all workspace packages are symlinked correctly; no peer dependency errors.

5. **Given** the repo is in its initial state **When** `git status` is checked **Then** all files are committed to an initial commit; `.gitignore` excludes `node_modules/`, `dist/`, `apps/web/public/vocab.json`.

## Dev Notes

### What This Story IS and IS NOT

**In scope (Story 1.1):**
- Turborepo + pnpm monorepo scaffold
- Package stubs: `package.json`, `tsconfig.json`, empty `src/index.ts` for `packages/schema`, `packages/story-loader`, `packages/typescript-config`, `packages/eslint-config`
- App stubs: minimal `package.json` for `apps/web`, `apps/api`
- Working `pnpm install` from root
- Initial git commit

**Explicitly NOT in scope for 1.1 (later stories):**
- `packages/schema` content: `story.v1.json`, TypeScript types, `tsup` build — **Story 1.2**
- `packages/story-loader` content: AJV validation, `v1.ts`, `LoaderError` — **Story 1.3**
- `apps/web` content: Vite, React, TypeScript, Tailwind, shadcn/ui, Zustand stores, design tokens — **Story 1.4**
- Full `turbo.json` pipeline: `build-vocab`, `test:unit`, `test:e2e`, `lint` tasks — **Story 1.5**
- GitHub Actions CI, ADRs, `CONTRIBUTING.md`, `apps/story-generator/` placeholder — **Story 1.5**

### Critical: `create-turbo@latest` Default Cleanup

`npx create-turbo@latest` creates default apps (`apps/docs`, `apps/web` with Next.js stubs, etc.) that MUST be removed. Replace with project-specific stubs. Do not keep any Next.js or default Turborepo app content.

The default `turbo.json` tasks (`build`, `dev`, `lint`, `check-types`) can remain as-is from the scaffold — Story 1.5 will overwrite with the full pipeline. Do not prematurely add `build-vocab`, `test:unit`, or `test:e2e` tasks in this story.

### pnpm-workspace.yaml — Exact Content Required

```yaml
packages:
  - "apps/web"
  - "apps/api"
  - "packages/*"
```

**`apps/story-generator` MUST NOT appear here.** It is a Python project, not a Node.js package, and is explicitly excluded from the pnpm workspace per architecture. `pnpm install` would fail or error if Python dependencies were included.

### Package Stub: `packages/typescript-config/`

This provides shared tsconfig bases. Minimal required files:

`package.json`:
```json
{
  "name": "@nihonnohon/typescript-config",
  "version": "0.0.0",
  "private": true,
  "license": "MIT",
  "exports": {
    ".": "./base.json",
    "./base.json": "./base.json",
    "./react-library.json": "./react-library.json"
  }
}
```

`base.json` — canonical tsconfig base for all packages:
```json
{
  "$schema": "https://json.schemastore.org/tsconfig",
  "compilerOptions": {
    "strict": true,
    "module": "ESNext",
    "moduleResolution": "bundler",
    "target": "ES2022",
    "lib": ["ES2022"],
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "skipLibCheck": true,
    "isolatedModules": true
  }
}
```

`react-library.json` — extends base, adds React/DOM:
```json
{
  "$schema": "https://json.schemastore.org/tsconfig",
  "extends": "./base.json",
  "compilerOptions": {
    "lib": ["ES2022", "DOM", "DOM.Iterable"],
    "jsx": "react-jsx"
  }
}
```

**Key:** `"moduleResolution": "bundler"` is the TypeScript 5.x standard for Vite + tsup builds. Do NOT use `"node16"` or `"nodenext"` — those require `.js` extensions on all imports.

### Root `package.json` Requirements

Must have `"private": true` and `turbo` in devDependencies. The scaffold provides this. Confirm it does NOT list workspace packages under `dependencies` — those are managed by `pnpm-workspace.yaml`.

```json
{
  "name": "nihonnohon",
  "private": true,
  "scripts": {
    "build": "turbo build",
    "dev": "turbo dev",
    "lint": "turbo lint",
    "typecheck": "turbo typecheck"
  },
  "devDependencies": {
    "turbo": "latest"
  },
  "packageManager": "pnpm@..."
}
```

### `.gitignore` Must Include

```
# dependencies
node_modules/

# build outputs
dist/
.turbo/

# generated files (never committed)
apps/web/public/vocab.json

# local environment
.env
.env.local
*.local
```

`apps/web/public/vocab.json` is generated by `scripts/build-vocab.ts` (Story 1.5) and must never be committed. Including it now prevents accidental commit later.

### Testing: No Vitest Tests in This Story

Story 1.1 has no unit tests. The `pnpm install && pnpm turbo typecheck` run is the only verification. Vitest is added in Story 1.4 (apps/web scaffold).

### No `tsup` in This Story

`tsup` builds for `packages/schema` and `packages/story-loader` are Story 1.2 and 1.3 scope respectively. Do NOT add tsup configuration or `build` scripts to package stubs in this story.

### Workspace Version Protocol

Use `"workspace:*"` for all cross-workspace package references (pnpm workspace protocol). Example: `"@nihonnohon/schema": "workspace:*"`.
