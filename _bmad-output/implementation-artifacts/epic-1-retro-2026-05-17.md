**Code review is finding real defects.** Both Story 1.2 and 1.3 reviews caught substantive bugs — not style nits. The BOM encoding fix (`utf-8-sig`), the test assertion tightening, and the `response.text` None guard were genuine issues that would have caused confusing failures downstream.

## Challenges

### Windows dev environment vs. POSIX-targeted tooling

This was the dominant cross-story friction point.

| Story | Issue | Fix applied |
|---|---|---|
| 1.2 | Makefile `/tmp` path broken on Windows | Replaced with `python3 -c` string prepend |
| 1.2 | `datamodel-codegen` binary not on PATH | Used `python3 -m datamodel_code_generator` |
| 1.3 | `PYTHONPATH=src` needed (package not installed) | Added to Makefile `spike` target |
| 1.3 | Windows console can't encode `✓` (cp1252) | Replaced with ASCII `OK` |
| 1.3 | `make dev` uses `lsof` (POSIX-only) | **Not yet surfaced — will hit in Epic 2** |

**Root cause:** Development environment is Windows; production target is POSIX/Cloud Run. Every Makefile recipe written with POSIX assumptions surfaces this.

### Gemini API surface surprises

- `response.text` returns `None` when Gemini blocks output (safety filter, MAX_TOKENS). `json.loads(None)` raises `TypeError` with no diagnostic — patched in code review.
- The `response_schema` approach with complex nested Pydantic models (`RootModel`, `Enum`) has limitations. Plain JSON mode is more reliable for the M0 spike; Story 2.2 should evaluate whether structured output adds value in the production agent.

## Key Insights

**1. Scope-bounding incremental delivery works.** Story 1.2's `validator.py` deliberately checked only required-field presence, deferring semantic validation to Story 2.2. This kept the story achievable and the interface stable. Epic 2 should continue this pattern.

**2. Three-layer validation (jsonschema + parallel-array + Pydantic) is necessary but not sufficient.** The LLM can produce semantically wrong content that passes all three structural checks (e.g., grammatically incorrect Japanese, out-of-level vocabulary). M2's agentic verification loop addresses this — it should not be cut from scope.

**3. deferred-work.md is accumulating actionable signal.** 8 items logged. Several are directly relevant to Epic 2 stories: markdown fence stripping, 60-second timeout enforcement, duplicate vocab ID handling. Review this file before writing Story 2.2 context.

**4. PYTHONPATH=src is a technical debt workaround.** Works for the spike but should not propagate into the production server entry point (Story 2.2). Install with `pip install -e .` or configure a proper `src` layout entry in `pyproject.toml`.
