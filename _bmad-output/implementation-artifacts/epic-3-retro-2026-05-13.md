## What Went Well

**Boundary discipline held across all four stories.**
The `ReaderRoute` component body was declared frozen in Epic 2 and never touched. Only the loader evolved across three stories (3.3, 3.4) — each time additively. No structural rework.

**End-to-end coherence at every story boundary.**
Each story delivered a genuinely usable increment: 3.1 gave the cards, 3.2 gave the filter, 3.3 made URLs real and shareable, 3.4 opened the app to community files. No story left the app in a broken intermediate state.

**Test trajectory healthy and consistent.**
83 → 94 → 126 → 130 → 139. Each story added meaningful tests; no story traded coverage for speed.

**Review process caught a genuine production edge case.**
Story 3.4's `saveStory` resolved on `request.onsuccess` instead of `tx.oncomplete`. If the IDB transaction aborted after the put (e.g. quota exceeded), the caller got a successful promise, `navigate` fired, and the user would hit a 410 with no explanation. Hard to reproduce locally; code review caught it cleanly.

**IndexedDB path resolution via HTTP status codes was a clean design call.**
Using status 404 for manifest miss and 410 for storage miss gave `ReaderError` two distinct, accurate messages without string parsing or new error types. Extensible to future error states.

**3-path loader composed without restructuring.**
Adding the IndexedDB fallback to the existing manifest-lookup loader was purely additive. No existing tests broke; the loader tests covered all three paths explicitly.

**RT's assessment:** "That's gone well, very happy."

## Challenges and Learnings

**React Router v6 test pattern still not in `project-context.md`.**
The Epic 2 retro explicitly called for documenting `vi.mock('react-router-dom') + MemoryRouter` before Story 3.1. It was used correctly across all four stories — but from retro memory, not a doc. We got lucky; the pattern is non-obvious to a developer arriving cold.

*Action:* Write it into `project-context.md` now, before Epic 4 Story 4.1.

**`renderLibrary` test helper silently overrode `useNavigate` mock.**
The Story 3.4 test helper set `vi.mocked(useNavigate).mockReturnValue(vi.fn())` unconditionally, clobbering the `mockNavigate` the test had just set. The valid-upload test passed 0 navigate calls. Caught during the test run, fixed by threading `navigate` as an optional parameter. Pattern to remember: shared render helpers that set mocks need an escape hatch for test-specific mock injection.

**FileReader mock pattern is new territory.**
Stubbing `new FileReader()` in jsdom — capturing `reader.onload` after `readAsText` fires, setting `mockFileReader.result`, calling `onload` manually inside `act()` — is non-trivial. Elena worked through it; worth documenting alongside the React Router pattern.

*Action:* Add FileReader mock pattern to `project-context.md` in the same pass as the React Router pattern.

**`FORMAT_SPEC_URL` is a placeholder hardcoded with a guessed GitHub username.**
LibraryRoute exports `FORMAT_SPEC_URL = 'https://github.com/rupertthomas/nihonnohon/...'`. Correct path, speculative username. Not a v1 blocker but must be updated before the app ships publicly.

*Action:* Deferred until the repo has a confirmed public URL.

**Story 3.2 discovered Radix UI not installed.**
The epics spec described a "Radix Select" for the difficulty filter. The dev agent correctly identified that `@radix-ui/react-select` was not in `package.json` and switched to native `<select>` styled with Tailwind — satisfying all ACs without a new dependency. This was the right call, but the spec/reality mismatch added friction.

## Process Agreements

- **Shared render helpers need mock-override escape hatches.** Any test helper that calls `vi.mocked(x).mockReturnValue(...)` must accept the mock as an optional parameter so individual tests can inject their own. Pattern: `function renderX(props = defaults, navigate = vi.fn())`.
- **New Web API patterns get documented immediately.** FileReader mock, IndexedDB test isolation, and any other non-obvious jsdom pattern should be added to `project-context.md` the moment they're worked out — not deferred to the retro.
- **React Router v6 test pattern is non-negotiable to document before Epic 4.** This has been deferred twice now.
